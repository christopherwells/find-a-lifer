## Key Files & Architecture

### Frontend
- `frontend/src/main.tsx` — App wrapped in `<LifeListProvider>`
- `frontend/src/App.tsx` — Root state: currentWeek, viewMode, heatmapOpacity, goalSpeciesCodes, selectedRegion, selectedSpecies, sidePanelCollapsed, goalBirdsOnlyFilter. Loads goal species from IndexedDB when viewMode or filter changes. Passes all state as props to SidePanel and MapView.
- `frontend/src/components/SidePanel.tsx` — **~4300+ lines. All tab components defined inline:** ExploreTab, SpeciesTab, GoalBirdsTab, TripPlanTab, ProgressTab, ProfileTab, plus SpeciesInfoCard. This file is the bulk of the app UI.
- `frontend/src/components/MapView.tsx` — MapLibre map, grid overlay rendering, all 4 heatmap view modes, click-to-inspect popups (lifers + goal birds), region flyTo zoom, opacity control. Uses module-level `speciesMetaCache` to avoid re-fetching species. Uses React refs (`goalSpeciesIdSetRef`, `viewModeRef`, `weeklyDataRef`, etc.) for access inside map click handlers.
- `frontend/src/contexts/LifeListContext.tsx` — IndexedDB CRUD: markSpeciesSeen/Unseen, toggleSpecies, importSpeciesList (source: 'import'), getTotalSeen, isSpeciesSeen. DB: `find-a-lifer-db` v2, store `lifeList`, keyPath `speciesCode`.
- `frontend/src/lib/goalListsDB.ts` — IndexedDB CRUD for goal lists: saveList (put), deleteList, renameList, addSpeciesToList, removeSpeciesFromList. Store `goalLists`, keyPath `id` (UUID). Active list ID in localStorage key `activeGoalListId`.

### Backend
- `backend/main.py` — FastAPI with CORS, endpoints: /api/health, /api/species, /api/weeks/{1-52}, /api/grid, /api/regions
- `backend/app_data/species.json` — 2490 species with fields: species_id, speciesCode, comName, sciName, familyComName, taxonOrder, conservStatus, difficultyScore, difficultyLabel, isRestrictedRange, invasionStatus, seasonalityScore, peakWeek, rangeShiftScore, photoUrl
- `backend/data/regions.geojson` — Regions: US Northeast, Southeast, West, Alaska, Hawaii
- Weekly data: ~62K records (winter) to ~117K records (spring peak), each: {cell_id, species_id, probability}
- Grid: 312 cells covering continental US

## Important Patterns & Gotchas

1. **SidePanel.tsx is massive (~4300+ lines)** — all tab components are inline functions, not separate files
2. **Module-level caches**: `speciesMetaCache` in MapView.tsx shared across overlay calculations
3. **Ref pattern for map handlers**: viewModeRef, weeklyDataRef, goalSpeciesCodesRef, seenSpeciesRef — because map click callbacks capture stale closures
4. **goalSpeciesIdSetVersion counter** (not boolean): Incremented to trigger re-renders after async ID set build
5. **createPortal for modals**: SpeciesInfoCard renders via `createPortal(jsx, document.body)` to escape overflow-y-auto z-index issues
6. **Curated suggestion arrays**: RAPTORS, OWLS_NIGHTBIRDS, COLORFUL_CHARACTERS, LBJS are hardcoded const arrays of eBird species codes
7. **Overlay useEffect dependency array** is extensive: [weeklyData, viewMode, heatmapOpacity, goalBirdsOnlyFilter, seenSpecies, selectedSpecies, legendMin, legendMax, goalSpeciesIdSetVersion]
8. **Compare mode infinite loop prevention**: `lastProcessedLocationRef` checks cellId against last processed to prevent re-routing same location
