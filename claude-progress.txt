- `frontend/src/components/GoalBirdsTab.tsx` — **2,365 lines.** Goal list CRUD, 11 suggestion categories with curated arrays.

- `frontend/src/components/SpeciesTab.tsx` — **698 lines.** Species checklist, search, filters, family groups.

- `frontend/src/components/ProgressTab.tsx` — **246 lines.** Stats dashboard, milestones, family breakdown.
- `frontend/src/components/ProfileTab.tsx` — **277 lines.** CSV import/export, life list stats, clear/reset.
- `frontend/src/components/types.ts` — **99 lines.** Shared interfaces (MapViewMode, Species, SelectedLocation, etc.).
- `frontend/src/contexts/LifeListContext.tsx` — IndexedDB CRUD: markSpeciesSeen/Unseen, toggleSpecies, importSpeciesList (source: 'import'), getTotalSeen, isSpeciesSeen. DB: `find-a-lifer-db` v2, store `lifeList`, keyPath `speciesCode`.

## Important Patterns & Gotchas

1. **SidePanel refactored** — tab nav shell (182 lines) imports 6 separate tab files + types.ts. GoalBirdsTab (2,365 lines) is now the largest component file
2. **Module-level caches**: `speciesMetaCache` in MapView.tsx shared across overlay calculations
3. **Ref pattern for map handlers**: viewModeRef, weeklyDataRef, goalSpeciesCodesRef, seenSpeciesRef — because map click callbacks capture stale closures
4. **goalSpeciesIdSetVersion counter** (not boolean): Incremented to trigger re-renders after async ID set build
5. **createPortal for modals**: SpeciesInfoCard renders via `createPortal(jsx, document.body)` to escape overflow-y-auto z-index issues
6. **Curated suggestion arrays**: RAPTORS, OWLS_NIGHTBIRDS, COLORFUL_CHARACTERS, LBJS are hardcoded const arrays of eBird species codes
7. **Overlay useEffect dependency array** is extensive: [weeklyData, viewMode, heatmapOpacity, goalBirdsOnlyFilter, seenSpecies, selectedSpecies, legendMin, legendMax, goalSpeciesIdSetVersion]
8. **Compare mode infinite loop prevention**: `lastProcessedLocationRef` checks cellId against last processed to prevent re-routing same location
